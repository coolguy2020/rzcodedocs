# RZCode 定价方案

本文为公开定价说明，适用于 RZCode 的 credits 购买与文档生成计费。

## 目录

## 1. 充值价格

- 0.99 元 = 1 credit

## 2. 计费范围

- 生成源代码文档：免费
- 生成软件使用手册（DOCX）：按 credits 计费
- 生成帮助填写软著的文档：0.1~0.4 credits

## 3. 软件使用手册计费规则

软件使用手册生成按任务消耗 credits，计费由三部分组成：

### Step 1：截图预处理

- 每张截图消耗 0.1 credits

### Step 2：截图与代码校对

#### 预扫描步骤

分析总结代码的内容和模块，提取关键信息：

- 当代码文件数量 <= 3000 时，0.1 credits
- 当代码文件数量 > 3000 时，0.2 credits

#### 截图校对

按项目代码文件总数分档计费（单位：credits / 张截图）：

| 代码文件数 | 单价 |
| --- | --- |
| <= 500 | 0.3 |
| 501-1000 | 0.5 |
| 1001-3000 | 0.7 |
| 3001-6000 | 0.9 |
| 6001-10000 | 1.0 |
| > 10000 | 不支持 |

### Step 3：文档内容生成

- 0.3 credits / 章节

章节计数口径：
- 顶层说明章节按章节数计费
- 功能章节按二级子章节数计费

## 4. 计费示例

若一次任务包含：
- 18 张截图
- 代码文件数 <= 500
- 共 9 个计费章节

则预计消耗：

```text
(0.1 + 0.3) x 18 + 0.1 + 0.3 x 9 = 10 credits
```

## 5. 使用限制

- 单个项目最多上传 50 张截图。可通过[企微联系客服](/contact)，提升单个项目上限。
- 当前暂不支持代码文件数超过 10000 的项目。

- 具体套餐与活动以正式发布的 [定价](/pricing) 页面为准
- 定价策略可能根据产品升级进行调整，调整后将提前公示
